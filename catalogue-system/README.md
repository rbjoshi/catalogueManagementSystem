# CatalogueHub — Enterprise Catalogue Management System

A full-stack catalogue management system built with **Spring Boot 3** + **React 18 + TypeScript**.

---

## Architecture

| Layer | Technology |
|---|---|
| Frontend | React 18, TypeScript, Vite, TailwindCSS, TanStack Query, dnd-kit |
| Backend | Spring Boot 3.2, Hibernate 6, Spring Security, JWT |
| Database | PostgreSQL 16 (via Flyway migrations) |
| Cache | Redis 7 |
| PDF | iText7 (async generation) |

### Enterprise ID Pattern
Every enterprise gets a unique ID with the format:
```
e-<32 random alphanumeric chars>
Example: e-aB3kL9mNqRsTuVwXyZ1234567890aB
```
Generated by `EntIdGenerator.generate()` in `com.catalogue.util`.

```bash
docker-compose up -d
```

- Java 17+


## Database Schema

All Flyway migrations are in `backend/src/main/resources/db/migration/`.

### Key Tables

| Table | Description |
|---|---|
| `enterprise` | Top-level tenant. `ent_id` = `"e-"` + 32 alphanumeric chars |
| `users` | Users scoped to an enterprise. Roles: OWNER / ADMIN / EDITOR / VIEWER |
| `item_type` | Normalised item type lookup (per enterprise) |
| `item_sub_type` | Sub-classification under a type |
| `item_size` | Size labels (XL, 250ml, A4 …) |
| `item_brand` | Brand / manufacturer lookup |
| `items` | Full item record with 25+ fields + JSONB images, tags, attributes |
| `catalogue_templates` | Reusable layout presets (Grid 2×3, Feature Showcase …) |
| `catalogues` | Catalogue with designer `layout_json` (JSONB) |
| `catalogue_items` | Junction: which items appear in which catalogue + per-slot overrides |
| `pdf_jobs` | Async PDF export job tracker |

## API Reference

All endpoints require `Authorization: Bearer <token>` except `/auth/*`.

### Auth
```
POST /api/auth/register   Register enterprise + owner
POST /api/auth/login      Login → JWT
```

### Items
```
GET    /api/items              List / search items (paginated)
POST   /api/items              Create item
GET    /api/items/:id          Get item
PUT    /api/items/:id          Update item
DELETE /api/items/:id          Soft delete
POST   /api/items/bulk         Bulk create
```

### Catalogues
```
GET    /api/catalogues              List catalogues
POST   /api/catalogues              Create catalogue
GET    /api/catalogues/:id          Get catalogue
PUT    /api/catalogues/:id          Update layout + items
POST   /api/catalogues/:id/publish  Publish
DELETE /api/catalogues/:id          Archive
POST   /api/catalogues/:id/export-pdf   Request PDF (async → returns job_id)
GET    /api/catalogues/pdf-jobs/:jobId  Poll PDF job status
GET    /api/catalogues/templates    List design templates
```

### Lookups
```
GET  /api/lookups             All lookups in one call
GET  /api/lookups/types
POST /api/lookups/types
GET  /api/lookups/sub-types?typeId=
POST /api/lookups/types/:id/sub-types
GET  /api/lookups/sizes
POST /api/lookups/sizes
GET  /api/lookups/brands
POST /api/lookups/brands
```

## Changing the ENT ID Pattern

Edit `com.catalogue.util.EntIdGenerator`:
```java
private static final String PREFIX = "e-";          // change prefix here
private static final int RANDOM_PART_LENGTH = 32;   // change length here
```

The DB column is `VARCHAR(35)` — adjust in the migration if you change length.
